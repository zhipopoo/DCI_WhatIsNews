## 2. 首页改造

- **Hero 走马灯**（`HeroCarousel.tsx`）：纯 CSS/React 实现，5 秒自动轮播，悬停暂停，导航圆点，渐隐过渡
- 移除静态 Hero section 和 Top Stories 卡片区（走马灯已承载）
- 隐藏导航栏 "Home" 链接，点击 Logo 返回首页

## 3. 富文本编辑器（TipTap）

- 选用 **TipTap**（MIT 免费许可证），替换 react-quill（空白 bug）和 TinyMCE（需 License）
- 支持功能：
  - 文字格式：粗体、斜体、下划线、删除线、H1-H3 标题
  - 列表：有序/无序列表、引用块、代码块、分割线
  - 图片：点击图标上传插入
  - 链接：添加/删除超链接
  - 视频：上传后嵌入 HTML5 video 播放器（自定义 `VideoNode` 扩展）
  - 音频：上传后嵌入 HTML5 audio 播放器（自定义 `AudioNode` 扩展）
  - 附件：PDF/Word/Excel/ZIP 等上传后插入为下载卡片
  - 表格：插入编辑表格
- 粘贴/拖拽图片自动上传

## 4. 媒体文件生命周期管理

- **删除文章** → 自动清理其引用的所有 `/uploads/` 文件（磁盘 + 数据库 `media_file` 记录）
- **编辑文章** → 对比新旧内容差异，自动清理被移除的文件
- **共享保护** → 多文章引用同一文件时保留不删
- 实现：`NewsServiceImpl.cleanUpMediaFiles()` / `cleanUpRemovedMediaFiles()`

## 5. Media Library 重构

- 卡片网格 → **表格视图**（序号、文件名/路径、类型、大小、引用数、日期、操作）
- 点击 "View" 展开子行，显示引用该文件的所有文章标题及编辑链接
- 后端新增 `GET /api/admin/media/{id}/references` 引用查询接口
- 上传按钮旁提示 `Max 2GB per file. For larger files, use chunked upload or object storage.`

## 7. 用户管理（Admin Panel）

- 侧边栏新增 **Users** 入口
- **用户列表**：表格展示用户名、显示名、邮箱、状态、最后登录时间
- **新建用户**：填写用户名、密码、显示名、邮箱
- **编辑用户**：修改信息、启用/禁用，密码留空则保持不变
- **删除用户**：默认 admin 不可删除
- **修改密码**：弹窗输入旧密码 + 新密码 + 确认，后端 BCrypt 验证
- 后端：`UserService` + `UserController` + `AdminUserDTO` + `CreateUserRequest` + `ChangePasswordRequest`

## 10. Bug 修复

- **Flyway checksum 不匹配** → V1 还原为原始内容 + V2 迁移策略调整（先 UPDATE 再 INSERT 避免 slug 冲突）
- **PostgreSQL 只读事务 UPDATE 报错** → `getPublishedNews()` 去掉 `@Transactional(readOnly = true)`
- **Docker 后端 volume 权限** → 容器以 root 运行，`/app/uploads` 可写入
- **导航栏重复 News 链接** → 动态分类 nav 中过滤掉 `slug === 'news'`
- **CI Docker Hub 不可达** → 所有 Dockerfile 和 `.gitlab-ci.yml` 镜像源切换为 `docker.m.daocloud.io`
